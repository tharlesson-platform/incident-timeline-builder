# Postmortem Template

O arquivo `postmortem-draft.md` e gerado com as seguintes secoes:

- metadata
- executive summary
- impact
- detection
- root cause hypothesis
- contributing factors
- what worked
- what failed
- 5 Whys snapshot
- follow-up actions
- evidence sources

## Como usar na pratica

1. gerar o draft com `incident-timeline-builder build`
2. revisar o incidente principal e confirmar a causa raiz
3. substituir `TBD` por owners reais
4. transformar os follow-ups em itens rastreaveis
